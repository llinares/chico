Getting started

Setting up your project with Chico UI.

1. Download

Getting started with Chico UI is easy. We recommend you to get the uncompressed files for your development/debugging stage. Use the compressed files for production releases.

Compressed (production)

JavaScript file, CSS file

Uncompressed (development)

Custom build

However, if you prefer to work with the latest version of Chico UI, you can build it by your own from its repository:

View on GitHub

2. Include

HTML

We hate writing CSS hacks, so we prefer to use IE's conditional comments. By using this code in your HTML, Chico UI will fix many visual inconsistencies:

<!DOCTYPE html>
<!--[if IE 8]><html class="no-js lt-ie10 lt-ie9 ie8" lang="en"><![endif]-->
<!--[if IE 9]><html class="no-js lt-ie10 ie9" lang="en"><![endif]-->
<!--[if gt IE 9]><!--> <html class="no-js" lang="en"><!--<![endif]-->

CSS

Make sure to add this into the tag <head>:

<link rel="stylesheet" href="chico.css"/>

In case that you need a tiny and powerful CSS grid system, we recommend you Mesh by Chico UI. Insert this metatag inside <head>. Doesn't matter if you call it before or after Chico UI.

JavaScript

Most of our framework’s functionality is built on jQuery. That’s why we need to pre-load this toolbox.

You can get jQuery from many places like Google CDN:

<script src="https://ajax.googleapis.com/ajax/libs/jquery/x.x.x/jquery.min.js"></script>

Or jQuery CDN:

<script src="http://code.jquery.com/jquery-x.x.x.min.js"></script>

After jQuery script call, you can load Chico UI:

<script src="build/chico-min-x.x.x.js"></script>

Don’t forget to link to asset files within the stylesheet code.

3. Create!

Chico UI is now setted up. Now, you can start creating a component.
